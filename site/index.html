Symbolic dimension tracking

Conformal doesn't just check concrete sizes. If you write zeros(n+1, 2*m), it tracks that as matrix[(n+1) x (2*m)] and carries those expressions through every operation. When a variable gets narrowed to a concrete value later (inside an if n == 5 branch, for example), the resolution propagates back through all the symbolic dimensions that depend on it.

This is what lets Conformal catch mismatches in code where sizes come from function arguments or size(A,1) calls, not just hardcoded numbers.

What it analyzes

Arithmetic, concatenation, indexing, struct fields, cell arrays, classdef objects, cross-file function calls, nested functions, closures, nargin/nargout, varargin. Loops use single-pass analysis or widening-based fixpoint iteration. An interval domain catches out-of-bounds indexing and division by zero. A relational pentagon domain tracks loop index bounds to suppress false positives on stencil patterns.

CI and verification

--format sarif outputs SARIF 2.1.0 with SHA-256 file hashes for audit traceability. --batch dir/ analyzes entire directories in one process. .conformal.json sets per-repo analysis modes. --coder checks for MATLAB Coder compatibility. Deterministic output: same file, same version, same result.
